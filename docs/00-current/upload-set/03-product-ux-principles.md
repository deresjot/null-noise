# Produkt- und UX-Prinzipien

## Entscheidungshilfe statt Analyse

`null-noise` hilft bei einer vorsichtigen, alltagstauglichen Entscheidung. Die App sagt nicht, wie ein Titel objektiv ist.

## Sichtbare Sprache

- eher ruhig
- eher wechselhaft/durchwachsen
- eher intensiv
- Metadaten deuten auf ...
- passt eher, wenn ...
- eher vorsichtig, wenn ...
- keine Szenenprüfung
- keine Entwarnung

## Nicht-Ziele

- keine Social Features
- keine Profile oder Konten
- kein Tracking
- keine algorithmische Personalisierung
- keine Scores, Prozentwerte oder Rankings
- keine scheinpräzisen Reizwerte
- keine Sicherheitsversprechen

## UX-Kern

- eine Hauptfrage pro Ansicht
- wichtigste Aussage zuerst
- kurze Sätze
- ehrliche Unsicherheit
- vorhersehbare Interaktion
- keine Hover-only-Hilfe
- Standard-UI ist der barrierearme Primärpfad

## Mobile App-Logik

- Home: kurzer Zweck, primäre Suche, kompakte Richtungsoptionen
- Suche: kompakter Steuerkopf, wiederholbare Ergebnisliste
- Detail: Lesart zuerst, Gründe und Grenzen danach
- Footer: Abschluss, kein zweiter Hero

## Umgang mit Komplexität

Neue Features werden daran gemessen, ob sie verständlicher, testbarer und ruhiger machen. Wenn ein Zusatzblock keinen klaren Entscheidungsnutzen bringt, bleibt er weg oder wird nachgeordnet.
